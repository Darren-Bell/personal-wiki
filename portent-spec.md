# Portent Spec

Open specification for work and personal knowledge bases by Luca Rossi.

Website: https://portent.md
Template: https://github.com/refactoringhq/portent-vault-template
Spec: https://github.com/refactoringhq/portent

## Core Concepts

- **8 types**: Project, Operation, Responsibility, Task, Event, Note, Topic, Person
- **PORT** (actionable) vs **ENTP** (context) split
- **2 relationships**: `belongs_to` (ownership) and `related_to` (association)
- **3 lifecycle states**: Capture → Organize → Archive

## Key Principles

- Convention over configuration
- Model reality before views
- Capture first, organize later
- Portable means simple
